Overview

The purpose of this project and codebase is to have a consistent schema across all WRI and WRI-related maps. This will aid in consistency of the experience for users as well as create an easier starting point for developers.

This framework provides default styling and some common mapping and interface functionality to incorporate into your projects. It provides a layout which includes default typography, space for the required branding and partner information, common map function buttons (home, zoom in, zoom out), an integrated search bar, and controls for defining map options.

Getting Started

The codebase lives at https://github.com/FriendlyDesignCo/wri-map-ui. You can clone this repository and begin customizing for your particular project. Should you find any issues, please note them in the core repository so we can always be updating our work.

Files Provided

  • index.html - A "kitchen sink" framework outline you can use as a starting point in your project. It includes demo elements of various types, and all of the default page elements.
  • wri-map.css - The framework's default stylesheet, defining the layout and all the elements used. This file has no external styling dependencies.
  • js/wri-map.js - The framework's default JavaScript library, which operates as a jQuery plugin. It implements the functionality necessary for all the built-in elements. Instructions for customization are available below.

Icons

This project integrates the Font Awesome library set for its use of icons. Use this set whenever possible to maintain consistency across all maps. Be sure to consult their full icon list before creating any other icons. You can learn more about Font Awesome here.

Headings and Branding

The heading area contains the following elements:

  • Title (to be placed as an "h1" in #map-title)
  • Subtitle (optional, to be placed under the main title)
  • Main branding (to be placed in #header-right as #partner-logo)
  • Navigation

Branding

This interface is designed to be primarily branded as part of one project or funding source at any given time. In other words, the logo area for main branding should be used for the primary brand. In the mobile version, this and all branding moves to the sidebar menu to save visual real estate.

There is an optional WRI "ribbon" that can be added at the top of the page. The code for this is to be placed directly above 'div#header-container':

<div id="ribbon">
A project convened by <a href="#partner" target="_blank">World Resources Institute</a>
</div>
<div id="header-container">...</div>

Any further branding (partners, co-funders, etc.) should be contained in the sidebar menu.

Site Navigation

Site navigation is defined as any links that do not manipulate the map and take the user to a different page. Only links that are site navigation should be put in the side menu. Conversely, the side menu is the only place the user should be able to find site navigation.

Typography & Colors

The framework uses two fonts, Open Sans Condensed and Helvetica.

Open Sans should be used for:

  • Page Headings (Bold)
  • Page Subtitles (Light)
  • Side Menu Site Navigation (Regular)
  • Titles in Modal Windows
  • Titles in Popups
  • Title Areas

Helvetica should be used for:

  • Buttons (Bold)
  • Map Controls (Regular)
  • Paragraph Text (Regular)
  • Side Menu Secondary Links
  • Side Menu Footer Text
  • Modal Window detail information
  • Form fields (search bars, form fields, etc.)
  • Labels, paragraphs, etc.

Colors

We use the standard colors of WRI throughout the experience.

  • WRI Orange (#f0ab00): Used sparingly
  • Light Grey (#dbdbdb): Side Menu activation button, disabled buttons
  • Medium Grey (#4c4c4c): Side Menu background
  • Dark grey (#3f3f3f): Map Controls background
  • Blue (#0692c3): Button Color
  • Dark Text (#231f20): Headlines, paragraph text

Map Controls

In order to standardize the experience across all maps, we have consolidated all of the functionality that effects the content viewed on the map into one area, the "Map Controls" section. All manipulation of the content on the map should only be contained in this menu.

The Map Controls area has been designed to contain as many menus and submenus as necessary for the desired project. It can accomodate the following form types:

  • Sub-menus
  • Sliders
  • Multiple Selections
  • Single Selections
  • Labels
  • Labels with thumbnails
